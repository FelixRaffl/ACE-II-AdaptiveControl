\section{Hardware realisation}

The hardware platform is an ESP32-WROOM module connected to an L298N H-bridge.
The L298N introduces an approximate voltage drop of \(\SI{4}{V}\), so with a
\(\SI{12}{V}\) supply the motor sees about \(\SI{8}{V}\). The motor is equipped
with a quadrature encoder. The sampled controller uses the period
\(T=\SI{0.08}{s}\), \(P(0)=100I\),
\(x_e(0)=\begin{bmatrix}-0.5&1.0\end{bmatrix}^{T}\), forgetting factor
\(\alpha=0.98\), desired polynomial \(q(z)=z^2-0.5z+0.06\), and a command
saturated at \(\pm255\).
\texttt{simulink/adaptive\_dcmotor.slx} is the first-order loop that runs on
the rig, while \texttt{simulink/adaptive\_dcmotor\_2nd\_hw.slx} is the
second-order diagnostic run reported in Subsection~\ref{sec:hardware-measurements}.

The controller is implemented by automatic code generation from Simulink using
the Embedded Real-Time (ERT) target, the ESP32 target support, and External
Mode for Monitor and Tune operation. Table~\ref{tab:esp32-pin-assignment}
gives the pin assignment.

The encoder resolution was determined by a one-revolution test as
\(44\) counts per revolution, corresponding to \(11\) encoder impulses with
x4 quadrature edge decoding. The speed conversion
\[
  \omega = \frac{2\pi \Delta N}{T\,\mathrm{CPR}}
\]
is therefore calibrated for \(\mathrm{CPR}=44\). The measured full-duty speed
of approximately \(\SI{780}{rad/s}\) corresponds to about \(\SI{7450}{rpm}\),
consistent with the gearbox-free 25GA371 speed range of roughly
\(\SIrange{3000}{8000}{rpm}\) and independently supporting the encoder scaling.
Figure~\ref{fig:hardware-rig} shows the assembled rig.

The encoder arithmetic in Table~\ref{tab:encoder-resolution} follows only from
the two measured constants \(\mathrm{CPR}=44\) and the full-duty speed
\(\SI{780}{rad/s}\); it is independent of any motor model. At \(T=\SI{2}{ms}\)
a moderate \(\SI{90}{rad/s}\) gives about \(1.3\) counts per sample, so the
speed signal degenerates towards a one-bit signal.

\subsection{Measurements}\label{sec:hardware-measurements}

The recorded hardware measurement is a closed-loop run of the second-order
adaptive controller from \texttt{simulink/adaptive\_dcmotor\_2nd\_hw.slx}. It
uses \(T=\SI{80}{ms}\), \(\alpha=0.98\), encoder scaling
\(2\pi/(0.08\cdot44)\), a square-wave reference between \(0\) and
\(\SI{780}{rad/s}\) with edges every \(\SI{4}{s}\), and a run length of
\(\SI{60}{s}\).

  \centering
  \includegraphics[width=0.9\linewidth]{../img/hw_response.pdf}
  \caption{Closed-loop response of the second-order adaptive controller on the
  hardware rig.}
  \label{fig:hardware-response}
\end{figure}

\begin{figure}[htbp]
  \centering
  \includegraphics[width=0.9\linewidth]{../img/hw_parameters.pdf}
  \caption{Second-order parameter estimates during the hardware run. The
  curves labelled \(a_1\) and \(a_2\) in the plot are the coefficients of
  \(y(k-1)\) and \(y(k-2)\), i.e. \(a_1\) and \(a_0\) in the notation of this
  report; the curves labelled \(b_1\) and \(b_2\) are the coefficients of
  \(u(k-1)\) and \(u(k-2)\), i.e. \(b_1\) and \(b_0\).}
  \label{fig:hardware-parameters}
\end{figure}

The response in Figure~\ref{fig:hardware-response} reaches and holds the upper
setpoint. That setpoint is the measured full-duty speed, so the actuator has
little headroom there. On falling edges the speed shows a limit cycle about
standstill: the driver cannot brake actively and the motor only coasts down.
The estimates in Figure~\ref{fig:hardware-parameters} never settle. They jump
at every reference edge and drift between edges; the estimate of the
coefficient multiplying \(y(k-2)\), the curve labelled \(a_2\) in the figure
and \(a_0\) here, hovers around \(+0.45\), whereas the exact ZOH value at
\(T=\SI{80}{ms}\) is numerically zero in
Table~\ref{tab:second-order-zoh-values}. This is the behaviour predicted by
the identifiability study in Section~\ref{sec:identifiability},
Table~\ref{tab:second-order-identifiability-study}.

Three mechanisms act here. The decisive one is encoder quantisation acting on
an ill-conditioned regression: that study isolates it with
open-loop, full-order PRBS data and no forgetting, so poor excitation and
forgetting are absent and the identification still fails. The excitation is
also weak, because the reference is a square wave and, between edges, \(u\)
and \(y\) are nearly constant, so information enters mainly at the edges.
Finally, with \(\alpha=0.98\) at \(T=\SI{80}{ms}\), the memory horizon is
\(1/(1-\alpha)=50\) samples, or \(\SI{4}{s}\), exactly the reference period;
what is learnt at one edge is therefore forgotten before the next. This run is
also a closed-loop identification, for which least squares is biased in
general; the open-loop study avoids that objection.
